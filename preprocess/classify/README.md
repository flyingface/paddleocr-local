# 分类服务 (Classify Service)

> 识别输入图像属于哪类业务文档,供 Gateway 做 OCR 路由决策。

## 状态

📝 **计划中** — 目录已建,等矫正服务跑通后开始。

## 设计要点

1. **L1 大类**(10 类,业务相关):身份证、护照、银行卡、发票、合同、报告、菜单、手写、其他
2. **L2 细类**(按需扩展):身份证正反面、增值税发票等
3. **模型选型**:倾向 PaddleClas(PP-LCNetV2),中文友好 + CPU 推理快
